### [SpatialWorld: Benchmarking Interactive Spatial Reasoning of Multimodal Agents in Real-World Tasks]
- 类型：论文
- 方向：多模态 / Agent
- 发布日期：2026-06-08
- 来源与作者：arXiv；Hongcheng Gao 等
- 一句话结论：论文结论是，当前顶级多模态 Agent 在真实交互式空间任务上的成功率仍然很低。
- 核心内容：作者提出 SpatialWorld，用统一协议整合 8 个异构仿真后端，构建 760 个人工标注任务，评测 Agent 在部分可观测、需要主动探索的真实空间任务中的表现。任务覆盖家居、出行、社交协作等场景，并提供参考轨迹与终态校验器。论文报告 GPT-5 的平均任务成功率仅 17.4%，领先开源模型 Qwen-3.5 为 14.1%。这说明“会看图回答”与“会在空间环境中持续行动”之间仍有明显鸿沟。
- 创新点：统一多后端交互评测协议；强调主动探索而非静态 VQA；把任务成功率与执行效率分开分析。
- 为什么值得关注：这是多模态 Agent 从“看懂”走向“进场操作”的关键评测基座，我判断它会成为空间智能和 embodied agent 的重要参照。
- 局限与疑点：目前仍是仿真环境；成功率低说明任务设计强，但也可能混入环境工程复杂度；暂未检索到公开代码仓库。
- 开源情况：暂未检索到明确开源仓库。
- 原始链接：https://arxiv.org/abs/2606.09669

### [iOSWorld: A Benchmark for Personally Intelligent Phone Agents]
- 类型：论文
- 方向：Agent
- 发布日期：2026-06-08（arXiv 版）
- 来源与作者：arXiv；Lawrence Keunho Jang, Mareks Woodside, Ruslan Salakhutdinov 等
- 一句话结论：论文结论是，手机 Agent 的真正难点不只是 GUI 操作，而是跨 App 的个性化记忆与用户建模。
- 核心内容：iOSWorld 构建了首个强调 persistent identity 的原生 iOS 模拟器基准，包含 26 个自建 App、133 个任务，覆盖单 App、多 App 和记忆/个性化三类。任务数据彼此关联，要求模型利用交易、消息、旅行和社交记录做决策。文中最优设置总体 52%，但多 App 仅 37%，说明 personal context reasoning 仍是主要瓶颈。论文还比较了 vision-only 与 vision+XML 两种设置。
- 创新点：把“个性化”正式纳入 mobile agent 基准；任务跨越 2 到 8 个 App；同时评估视觉与可访问性树输入。
- 为什么值得关注：我认为它补上了 phone agent 评测最缺的一块，即长期用户状态与偏好推理，而不是纯点击自动化。
- 局限与疑点：仍是模拟器；隐私、权限与真实设备延迟等问题没有完全体现；论文称将开源，但暂未检索到独立仓库链接。
- 开源情况：论文称 benchmark、apps、seed data、rubrics 与 eval code 将开源。
- 原始链接：https://arxiv.org/abs/2606.09764

### [OmniGameArena: A Unified UE5 Benchmark for VLM Game Agents with Improvement Dynamics]
- 类型：论文
- 方向：多模态 / Agent
- 发布日期：2026-06-08（arXiv 版）
- 来源与作者：arXiv；Mingxian Lin 等
- 一句话结论：论文结论是，游戏 Agent 不应只看一次性分数，还应衡量反思后能否持续改进。
- 核心内容：作者基于 Unreal Engine 5 搭建 12 个新游戏，覆盖 Solo、PvP 与 Coop，并设计统一动作接口。更重要的是提出 Improvement Dynamics Curve，用带工具的反思器在多轮中自动改写 skill prompt，观察分数随轮次变化以及对变体任务的泛化。这样把冷启动能力、反思改进能力和迁移能力拆开评测。相比传统单轮 leaderboard，这更贴近真实 agentic learning。
- 创新点：新增 Improvement Dynamics Curve；统一商业模型、开源模型和专用策略的评测口径；覆盖对抗与协作玩法。
- 为什么值得关注：它把 Agent 评测从“静态能力”推进到“能否自我改进”，这对游戏、软件操作和通用智能都很关键。
- 局限与疑点：游戏环境对现实任务的外推有限；反思增益可能受提示工程影响较大。
- 开源情况：已检索到 Hugging Face 数据集页面：https://huggingface.co/datasets/mxlin043/OmniGameArena
- 原始链接：https://arxiv.org/abs/2606.09826

### [SIGA: Self-Evolving Coding-Agent Adapters for Scientific Simulation]
- 类型：论文
- 方向：Agent
- 发布日期：2026-06-08
- 来源与作者：arXiv；Matthew Ho 等
- 一句话结论：论文结论是，通用 coding agent 要真正进入科研软件流程，关键不只是模型更强，而是补齐“工具接口契约”。
- 核心内容：SIGA 为科学仿真器加入轻量 adapter，把词汇、结构约束、验证规则和终止条件显式提供给 agent，并结合 retrieval、procedural memory 与 trajectory 内验证。作者主要在 GEOS 上评估，称完整 deck 生成约 5 分钟完成，TreeSim 超过 0.90；在更难集合上也明显优于裸 agent，并展示向 OpenFOAM、LAMMPS 的迁移。其核心思想是让 agent 学会“如何正确接入复杂软件”，而不是只会写代码。
- 创新点：把 simulator grounding 抽象成 adapter 层；引入 validation-enforced termination；支持从过往轨迹自演化更新 adapter。
- 为什么值得关注：我认为这是 Agent 落地工业/科研软件的高价值方向，实际意义不低于继续堆通用基准分数。
- 局限与疑点：当前验证集中在少数仿真器；指标 TreeSim 与真实科研产出之间仍有距离；暂未检索到明确开源链接。
- 开源情况：暂未检索到明确开源仓库。
- 原始链接：https://arxiv.org/abs/2606.09774

### [EVA-Bench Data 2.0: 3 Domains, 121 Tools, 213 Scenarios]
- 类型：技术博客
- 方向：Agent
- 发布日期：2026-06-04
- 来源与作者：Hugging Face / ServiceNow-AI；Tara Bogavelli 等
- 一句话结论：博客结论是，企业语音 Agent 的难点高度依赖领域知识、认证流程与策略约束，通用对话能力远远不够。
- 核心内容：EVA-Bench 从单领域扩展到 Airline CSM、ITSM、Healthcare HRSD 三个企业场景，共 213 个场景、121 个工具。作者详细披露了用 SyGra + GPT-5.4 联合生成 user goal、初始数据库与目标终态，再经结构校验、LLM 一致性校验和人工复核的流程。数据还特意覆盖多意图、不可满足请求、认证与对抗场景。它比常见 agent benchmark 更接近真实企业流程。
- 创新点：联合生成目标、数据库与终态；将认证和策略遵循作为核心评测维度；公开具体数据构建与验证流程。
- 为什么值得关注：如果你关心 voice agent、enterprise agent 或 tool-use reliability，这套数据的参考价值很高。
- 局限与疑点：主要是英语和企业流程；博客披露详尽，但不同模型在真实语音链路中的表现仍需独立复现。
- 开源情况：数据集开放：https://huggingface.co/datasets/ServiceNow-AI/eva-bench
- 原始链接：https://huggingface.co/blog/ServiceNow-AI/eva-bench-data

## 今日趋势观察
过去一周最强信号不是“更大的模型”，而是更真实、更长程、更强调交互闭环的 Agent 评测。无论是 SpatialWorld、iOSWorld 还是 OmniGameArena，都在把评测从静态问答推进到主动探索、跨应用记忆与多轮改进。另一个趋势是，工程接口本身正在成为研究对象，SIGA 和 EVA-Bench 都说明 tool contract、验证机制和策略约束决定了 Agent 能否落地。接下来最值得持续跟踪的方向，是个性化 mobile agent、空间智能 benchmark，以及面向复杂软件系统的 grounding / verification 层。

## 今日必读
1. SpatialWorld：它最直接暴露了多模态 Agent 在真实空间任务中的能力天花板，适合作为判断“空间智能是否真的进步”的新基线。
2. iOSWorld：它把 personalization 正式推到 phone agent 的核心位置，对未来的 mobile assistant、computer use 和长期记忆研究都很关键。
