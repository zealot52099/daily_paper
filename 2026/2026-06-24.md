### OpenThoughts-Agent: Data Recipes for Agentic Models
- 类型：论文
- 方向：Agent
- 发布日期：2026-06-23
- 来源与作者：arXiv；Negin Raoof、Richard Zhuang、Marianna Nezhurina、Etash Guha、Alex Dimakis、Ludwig Schmidt、Benjamin Feuer 等
- 一句话结论：公开、可复现实验驱动的 Agent 训练数据配方开始成为提升开源 Agent 模型的关键变量。
- 核心内容：论文指出，现有 SWE-Smith、SERA、Nemotron-Terminal 等开放数据多面向单一 benchmark，难以解释“泛化型 Agent 数据”如何构建。作者做了 100 多组 ablation，分析任务来源、多样性和流水线各阶段对 Agent 能力的影响，并整理 100K 训练样本微调 Qwen3-32B。原文报告该模型在 7 个 Agent benchmark 上平均准确率 44.8%，比 Nemotron-Terminal-32B 的 40.9% 高 3.9 个百分点；训练数据在不同规模下也表现出较好的 scaling。
- 创新点：开放数据构建流水线；系统性数据配方消融；同时发布训练集、实验数据和模型。
- 为什么值得关注：Agent 训练正从“模型结构技巧”转向“可验证数据工程”，对开源 Agent 追赶闭源模型很重要。
- 局限与疑点：平均 44.8% 说明距离稳健可用仍远；结论依赖所选 7 个 benchmark，真实软件环境泛化还需验证。
- 开源情况：论文称训练集、数据流水线、实验数据和模型开放，入口为 http://openthoughts.ai 。
- 原始链接：https://arxiv.org/abs/2606.24855

### SAFARI: Scaling Long Horizon Agentic Fault Attribution via Active Investigation
- 类型：论文
- 方向：Agent
- 发布日期：2026-06-23
- 来源与作者：arXiv；Chenyang Zhu、Jiayu Yao、Kushal Chawla、Youbing Yin、Nathan Wolfe 等；AIWILD @ ICML 2026
- 一句话结论：长轨迹 Agent 调试不能只靠把日志塞进上下文，需要工具化搜索和短期记忆式诊断循环。
- 核心内容：论文关注多步、多 Agent 任务失败后的 fault attribution，指出完整轨迹常超过上下文窗口，线性加载会导致 attention dilution。SAFARI 用工具增强诊断循环读取、搜索轨迹片段，并用 Short-Term Memory 跨轮保留推理状态。原文称在 1M token 预算下 Who&When 数据集提升 20%，在 25K token 预算下 TRAIL GAIA 子集提升 19%；当目标故障在模型原生上下文外 5 倍时仍保持 0.58 precision。
- 创新点：主动调查式轨迹诊断；轨迹搜索工具箱；面向超长执行日志的短期记忆机制。
- 为什么值得关注：Agent 落地后的核心问题不是单次成功率，而是失败能否定位、复现和修复，这篇切中工程痛点。
- 局限与疑点：结果仍主要是离线 benchmark；真实生产系统中的日志噪声、权限边界和隐私约束未必覆盖。
- 开源情况：arXiv 摘要未见明确代码链接。
- 原始链接：https://arxiv.org/abs/2606.24626

### GUI vs. CLI: Execution Bottlenecks in Screen-Only and Skill-Mediated Computer-Use Agents
- 类型：论文
- 方向：Agent
- 发布日期：2026-06-22
- 来源与作者：arXiv；Xiao Zhou、Siyue Zhang、Yilun Zhao、Jinbiao Wei、Tingyu Song、Arman Cohan、Chen Zhao
- 一句话结论：Computer Use 的 GUI 与 CLI 差异不能简单理解为“哪种接口更强”，瓶颈分别在视觉交互可靠性和技能覆盖率。
- 核心内容：作者构建了 440 个桌面任务、18 个应用、12 类工作流的 matched execution-layer benchmark，控制任务目标、初始状态和最终 verifier，使 GUI 与 CLI Agent 在相同目标上比较。原文报告最强 screen-only GUI Agent full pass rate 为 59.1%，强于原始 skill-mediated CLI 的 48.2%。但用 verifier-guided skill augmentation 后，CLI 成功率升至 69.3%，说明 CLI 短板很大程度来自技能接口不完整。
- 创新点：匹配式 GUI/CLI 评测设计；把执行接口与任务差异解耦；量化 skill coverage 对 CLI Agent 的影响。
- 为什么值得关注：对浏览器/桌面 Agent 产品设计有直接启发：与其争论 GUI 或 CLI，不如设计可扩展技能层和可靠 verifier。
- 局限与疑点：桌面任务集合是否代表真实企业软件流程仍需观察；技能增强可能引入较高人工或自动生成成本。
- 开源情况：arXiv 摘要未见明确代码链接。
- 原始链接：https://arxiv.org/abs/2606.24551

### Agentic Collaborative Cognition for Zero-Shot 3D Understanding
- 类型：论文
- 方向：两者相关
- 发布日期：2026-06-23
- 来源与作者：arXiv；Wenxin Wang、Bo Zhang、Feng Chen、Zixuan Wang、Wen Li、Changsheng Li、Yinjie Lei；ECCV 2026
- 一句话结论：把 3D 理解拆成规划 Agent 与感知 Agent 的闭环协作，可缓解有限视角视频带来的空间理解缺口。
- 核心内容：论文将 zero-shot 3D understanding 转化为多视角/关键帧理解问题，但指出现有方法受限于有限观察视角和隐式 3D 感知。方法用 Planning Agent 规划查询相关视角并补充缺失视角，用 Perception Agent 维护结构化 holistic cognitive map，跨视角分配一致实例 ID 并过滤候选。原文称在 6 个 benchmark 达到 SOTA，包括 ScanRefer Acc@0.5 提升 11.1%、3D-assisted dialog BLEU-1 提升 14.6、SQA3D EM 提升 2.1。
- 创新点：多 Agent 3D 场景认知图；查询驱动视角规划；跨视角实例一致性维护。
- 为什么值得关注：这是多模态 Agent 与空间智能结合的清晰范式，适合机器人、AR 和具身导航继续跟踪。
- 局限与疑点：依赖关键视角生成和候选过滤质量；真实动态场景、遮挡和传感器噪声下表现未知。
- 开源情况：arXiv 页面标注 project page：https://zhangbo135.github.io 。
- 原始链接：https://arxiv.org/abs/2606.24649

### DiffusionBench: On Holistic Evaluation of Diffusion Transformers
- 类型：论文
- 方向：多模态
- 发布日期：2026-06-23
- 来源与作者：arXiv；Xingjian Leng、Jaskirat Singh、Zhanhao Liang、Ethan Smith、Martin Bell、Aninda Saha、Yuhui Yuan、Liang Zheng
- 一句话结论：只看 ImageNet class-conditional FID 已不足以衡量 DiT 进展，T2I 应进入基础评测闭环。
- 核心内容：论文批评 DiT 研究过度收敛到 ImageNet class-conditional generation，可能无法反映真实文生图能力。作者提出 NanoGen 统一训练评测框架，称只需 12 行配置变化即可从 ImageNet 切到 text-to-image，并支持 RAE、VAE、pixel-space、MeanFlow 等方法。在 21 个 latent diffusion models 上，ImageNet 与 T2I 排名相关性很弱，Pearson 在 -0.377 到 -0.580 之间，因此建议用 DiffusionBench 同时报告两类任务。
- 创新点：统一 DiT 训练/评测框架 NanoGen；ImageNet 与 T2I 相关性实证分析；DiffusionBench 综合评测建议。
- 为什么值得关注：生成模型评测正从单指标转向任务覆盖，能减少“刷 ImageNet FID”的误导。
- 局限与疑点：摘要未说明人评或偏好评测覆盖；T2I prompt 多样性和安全维度仍可能不足。
- 开源情况：arXiv 摘要未见明确代码链接。
- 原始链接：https://arxiv.org/abs/2606.24888

### FLUX3D: High-Fidelity 3D Gaussian Generation with Diffusion-Aligned Sparse Representation
- 类型：论文
- 方向：多模态
- 发布日期：2026-06-23
- 来源与作者：arXiv；Haorui Ji、Weizhe Liu、Hongdong Li、Hengkai Guo
- 一句话结论：高质量 image-to-3DGS 的关键在于让 2D diffusion 表征与稀疏 3D voxel latent 更好对齐。
- 核心内容：论文指出，当前基于 sparse voxel 的 image-to-3D Gaussian Splatting 方法存在两个瓶颈：语义型 2D 特征会损失重建细节，标准 DiT 缺少密集 2D token 与稀疏 3D latent 对齐机制。FLUX3D 提出 Diffusion-Aligned Structured Latents（DA-SLAT）和 decoder-only 结构提升重建保真度，并设计 Sparse-structure Multimodal Diffusion Transformer（SMDiT）与 Modal-Aware RoPE（MARoPE）实现 2D-3D 对齐。原文称在 benchmark 上显著优于现有 SOTA，特别是外观 fidelity。
- 创新点：DA-SLAT 表征；SMDiT 稀疏结构感知扩散；MARoPE 跨模态位置编码。
- 为什么值得关注：3D asset 生成正在从几何可用转向纹理和细节保真，对游戏、仿真和空间内容生产有实际价值。
- 局限与疑点：摘要未披露速度、显存和失败案例；单图到 3D 的背面幻觉与物理一致性仍需细看。
- 开源情况：arXiv 摘要未见明确代码链接。
- 原始链接：https://arxiv.org/abs/2606.24874

## 今日趋势观察
今天的高价值内容集中在两个方向：一是 Agent 从“会做任务”转向“如何训练、如何诊断、如何评测执行层瓶颈”；二是多模态生成从单一指标转向更贴近真实使用的综合评测和 3D 表征对齐。Agent 领域尤其值得持续跟踪数据配方、长轨迹可观测性、Computer Use 技能层和 verifier 设计。多模态侧则建议关注 T2I/DiT 评测范式、image-to-3DGS、以及多 Agent 空间认知，这些方向与具身智能和内容生产的交叉越来越明显。

2. GUI vs. CLI：建议所有做 Computer Use/桌面 Agent 的团队阅读，它把 GUI grounding 与 CLI skill coverage 的瓶颈拆开，结论对产品架构选择很实用。
