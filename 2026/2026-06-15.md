### Gaze Heads: How VLMs Look at What They Describe
- 类型：论文
- 方向：多模态
- 发布日期：2026-06-12
- 来源与作者：arXiv；Rohit Gandikota、David Bau
- 一句话结论：少量可识别的 VLM attention heads 会追踪当前描述区域，并可作为无需训练的视觉注意控制杠杆。
- 核心内容：作者用漫画的空间叙事顺序定位“gaze heads”，再通过 attention mask 干预验证因果作用。论文报告，干预不足 9% 的 heads，可将回答导向指定画格，准确率达 83.1%；该现象还扩展到 COCO 图像及 2B–32B 多种模型。
- 创新点：以机制可解释性定位跨模态 grounding；用因果干预而非相关性热图验证；支持生成中途动态切换目标区域。
- 为什么值得关注：它把 VLM 内部解释转化为可操作的 inference-time control，对可控生成和安全审计都有直接价值。
- 局限与疑点：部分 frozen-encoder 架构未出现同类机制；漫画到复杂视频、密集场景的泛化仍待验证。
- 开源情况：代码、交互 Demo、数据已开放：[项目页](https://gaze.baulab.info/)
- 原始链接：https://arxiv.org/abs/2606.14703

### AgentSpec: Understanding Embodied Agent Scaffolds Through Controlled Composition
- 类型：论文
- 方向：Agent
- 发布日期：2026-06-12
- 来源与作者：arXiv；Jixuan Chen、Jianzhi Shen、Haoqiang Kang 等
- 一句话结论：Agent 模块的兼容性和交互效应，往往比单个 reasoning、memory 或 reflection 模块的独立强度更重要。
- 核心内容：AgentSpec 将 perception、memory、reasoning、reflection、action 与 learning 表示为带类型接口的可组合组件。作者在 DeliveryBench、ALFRED、MiniGrid、RoboTHOR 上做受控替换实验，发现多粒度记忆有助于长程状态跟踪，reflection 则存在纠错收益与成本的权衡。
- 创新点：标准化 Agent scaffold 接口；支持模块级因果比较；覆盖多种具身环境和模型骨干。
- 为什么值得关注：为 Agent 架构研究提供比“端到端排行榜”更可复现的实验方法，也有利于工程选型。
- 局限与疑点：结论主要来自模拟具身任务，尚不能直接外推到开放 Web、桌面 Computer Use 或生产工具链。
- 开源情况：代码、基线与 playground 已开放：[项目页](https://agentspec-embodied.github.io/)
- 原始链接：https://arxiv.org/abs/2606.14674

### Towards Direct Latent-Space Synthesis for Parallel Branches in LLM-Agent Workflows
- 类型：论文
- 方向：Agent
- 发布日期：2026-06-12
- 来源与作者：arXiv；Shikun Liu、Mufei Li、Dongqi Fu 等
- 一句话结论：Parallel-Synthesis 直接合并并行 worker 的 KV cache，可避免文本拼接并显著降低综合阶段延迟。
- 核心内容：方法用 cache mapper 校准独立分支的 KV cache，并用 adapter 让 synthesizer 接受非顺序缓存。论文报告在数学、科学问答、代码、GAIA 和数据库诊断等 9 个数据集上，7 个达到或超过文本拼接方案，TTFT 降低 2.5–11 倍。
- 创新点：将并行 Agent 分支保留为 latent cache；面向非顺序上下文训练 synthesizer；直接优化多 Agent 汇总的 prefill 开销。
- 为什么值得关注：这是 Agent 系统从提示词编排走向模型运行时原生接口的重要尝试，潜在价值集中在低延迟和长上下文成本。
- 局限与疑点：需要模型级 cache 接口和额外训练；跨模型、量化、分布式 serving 的兼容性尚不明确。
- 开源情况：论文摘要未声明公开代码或模型。
- 原始链接：https://arxiv.org/abs/2606.14672

### OmniVideo-100K: A Dataset for Audio-Visual Reasoning through Structured Scripts and Evidence Chains
- 类型：论文
- 方向：多模态
- 发布日期：2026-06-12
- 来源与作者：arXiv；Xinyue Cai、Chaoyou Fu、Yi-Fan Zhang、Ran He、Caifeng Shan
- 一句话结论：以实体锚定脚本和证据链生成音视频 QA，可显著增强跨片段、跨模态推理训练信号。
- 核心内容：数据引擎先生成包含全局实体表和分段音视频描述的 structured scripts，再挖掘跨片段 clues 生成 QA。由此构建 OmniVideo-100K 与人工核验的 OmniVideo-Test；作者报告微调 VITA-1.5、Qwen2.5-Omni-7B、Qwen3-Omni-30B 后，在自建测试集最高提升 20.59%，外部基准最高提升 12.64%。
- 创新点：用全局实体先验保持跨段指代一致；先证据挖掘再生成问题；同时覆盖音频、视觉与长时序关系。
- 为什么值得关注：高质量音视频 reasoning 数据仍稀缺，该流水线具有可扩展性。
- 局限与疑点：自动生成标注可能继承教师模型偏差；自建测试集上的大幅收益需结合数据泄漏检查理解。
- 开源情况：摘要未给出明确数据或代码链接。
- 原始链接：https://arxiv.org/abs/2606.14702

### Memento: Reconstruct to Remember for Consistent Long Video Generation
- 类型：论文
- 方向：多模态
- 发布日期：2026-06-12
- 来源与作者：arXiv；Xuan Wei、Longbin Ji、Guan Wang 等
- 一句话结论：让历史 memory 能重建主体，可将长视频身份保持从隐式条件转化为显式训练目标。
- 核心内容：Memento 联合训练逐镜头生成与 memory-based subject reconstruction，并以 dual-query memory 分离长期身份检索和短期关键帧连续性。作者称其在长期主体一致性、跨镜头连贯性和视觉质量上达到 SOTA。
- 创新点：以可重建性约束身份记忆；双查询解耦长期身份与短期运动；使用无代词主体描述提供精确监督。
- 为什么值得关注：主体漂移是长视频生成的核心瓶颈，该方法给出了清晰且可检验的 memory 设计原则。
- 局限与疑点：摘要未披露具体基准差值；多主体交互、遮挡和复杂镜头语言下的稳定性仍需核验。
- 开源情况：摘要未声明公开代码、模型或数据。
- 原始链接：https://arxiv.org/abs/2606.14667

### AdaSR: Adaptive Streaming Reasoning with Hierarchical Relative Policy Optimization
- 类型：论文
- 方向：两者相关
- 发布日期：2026-06-12
- 来源与作者：arXiv；Junlong Tong、Wenqi Xu、Yingqi Fan 等
- 一句话结论：AdaSR 让模型在音视频等流式输入到达时动态决定何时思考、投入多少计算。
- 核心内容：框架把 reasoning 分为 streaming reasoning 与输入结束后的 deep reasoning。HRPO 分阶段分配 advantage，并结合格式、准确率和自适应思考奖励；作者报告其相较 SFT 在准确率、计算效率和流式延迟间取得更优平衡。
- 创新点：将 test-time compute allocation 引入流式推理；分层 policy optimization；显式优化延迟与答案质量。
- 为什么值得关注：实时多模态 Agent 不能等待完整上下文，该工作直接面向持续感知和在线决策。
- 局限与疑点：摘要未给出绝对延迟和硬件条件；奖励设计在开放式任务上的稳健性仍待验证。
- 开源情况：代码已开放：[GitHub](https://github.com/EIT-NLP/StreamingLLM/tree/main/AdaSR)
- 原始链接：https://arxiv.org/abs/2606.14694

## 今日趋势观察
本期最明显的趋势是“结构显式化”：VLM 将视觉 grounding 映射到特定 heads，视频模型把身份保持写成重建目标，Agent 则把 scaffold 和并行分支变成标准接口或 latent cache。研究重点正从单纯扩大模型，转向控制内部机制、管理长期记忆和降低运行时成本。值得持续跟踪的方向包括 VLM 因果可解释性、流式多模态推理、Agent cache-native runtime，以及可组合 scaffold 的跨任务泛化。

## 今日必读
1. **Gaze Heads**：机制发现、因果验证和实际控制形成完整闭环，是少见的“解释即可用”工作。
2. **AgentSpec**：它提供了系统研究 Agent 模块交互的共同语言，可能比新增一个复杂 scaffold 更具长期基础设施价值。
