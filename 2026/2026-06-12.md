### SpatialClaw: Rethinking Action Interface for Agentic Spatial Reasoning
- 类型：论文 / 开源项目
- 方向：两者相关
- 发布日期：2026-06-11
- 来源与作者：NVIDIA、KAIST；Seokju Cho、Ryo Hachiuma、Abhishek Badki 等
- 一句话结论：持久化 Python 内核比一次性代码或固定 Tool Call 更适合开放式 3D/4D 空间推理。
- 核心内容：Agent 每轮执行一个代码单元，并保留深度、分割、相机几何及可视化状态。论文在 20 个空间基准上报告 59.9% 平均准确率，较近期空间 Agent 高 11.2 个百分点；同一配置迁移到六个 VLM backbone，无需专项训练。
- 创新点：持久化执行状态；基于中间视觉证据修正推理；自由组合 perception 与科学计算工具。
- 为什么值得关注：证明动作接口设计本身可以显著影响 Agent 上限。
- 局限与疑点：仍受 VLM、Depth Anything 3 和 SAM3 感知误差限制；多轮代码执行增加成本与安全风险。
- 开源情况：论文页面标注 Code/Project Page，具体仓库尚待索引稳定后复核。
- 原始链接：https://arxiv.org/abs/2606.13673

### HyperTool: Beyond Step-Wise Tool Calls for Tool-Augmented Agents
- 类型：论文
- 方向：Agent
- 发布日期：2026-06-11
- 来源与作者：上海交通大学、IQuest Research、北航；Yaxin Du、Yifan Zhou、Siheng Chen 等
- 一句话结论：将确定性的多工具流程折叠进一个可执行 MCP-style 调用，可减少上下文膨胀和推理碎片化。
- 核心内容：HyperTool 在代码块内调用原始工具 schema、传递中间值，仅向主推理轨迹返回聚合结果。在 MCP-Universe 上，Qwen3-32B 准确率从 15.69% 提至 35.29%，Qwen3-8B 从 9.93% 提至 33.33%。
- 创新点：改变模型可见的执行粒度；保持 MCP 兼容；对训练轨迹执行验证和证据一致性过滤。
- 为什么值得关注：可直接改善 Agent 延迟、上下文成本和复杂工具编排能力。
- 局限与疑点：动态代码带来 RCE 风险；隐藏中间状态削弱可解释性；过早作答成为新的主要错误。
- 开源情况：正文未提供可核验代码链接。
- 原始链接：https://arxiv.org/abs/2606.13663

### InterleaveThinker: Reinforcing Agentic Interleaved Generation
- 类型：论文 / 开源项目
- 方向：两者相关
- 发布日期：2026-06-11
- 来源与作者：CUHK MMLab、Meituan；Dian Zheng、Harry Lee、Manyuan Zhang、Hongsheng Li 等
- 一句话结论：Planner-Generator-Critic 流程可让冻结图像生成器完成长序列图文交错生成。
- 核心内容：Planner 预先生成全局步骤，Critic 检查每步图像并改写提示词。作者构建三个 SFT/RL 数据集，并以单步 GRPO 近似长轨迹优化；在 FLUX.2-klein 上，WISE 从 0.47 升至 0.73，RISE 从 13.3 升至 28.9。
- 创新点：规划与视觉反馈解耦；逐步纠错双奖励；兼容不同冻结图像生成器。
- 为什么值得关注：展示了通过 Agent 编排增强现有多模态生成器的可行路径。
- 局限与疑点：数据大量依赖闭源模型；未知概念无法通过 Critic 修复；长轨迹调用成本较高。
- 开源情况：[GitHub](https://github.com/zhengdian1/InterleaveThinker)、[Hugging Face](https://huggingface.co/InterleaveThinker)
- 原始链接：https://arxiv.org/abs/2606.13679

### WEAVER, Better, Faster, Longer: An Effective World Model for Robotic Manipulation
- 类型：论文 / 开源项目
- 方向：多模态
- 发布日期：2026-06-11
- 来源与作者：Arnav Kumar Jain、Yilin Wu、Jesse Farebrother、Gokul Swamy、Andrea Bajcsy
- 一句话结论：多视角 flow-matching 世界模型兼顾机器人预测保真度、长程一致性和速度。
- 核心内容：WEAVER 联合预测未来 latent 与 reward。论文报告模拟结果与真实成功率相关系数为 0.870；用于策略改进时，在 π0.5 上成功率提高 38%，测试时规划提高 14%，速度比既有世界模型快 5–10 倍。
- 创新点：多视角 latent 建模；联合未来状态与奖励；统一支持策略评估、改进和规划。
- 为什么值得关注：世界模型正从视频预测展示转向真实机器人闭环收益。
- 局限与疑点：结果对任务范围及基线设置敏感，长尾接触和分布外稳定性仍需验证。
- 开源情况：[项目页](https://arnavkj1995.github.io/WEAVER/)
- 原始链接：https://arxiv.org/abs/2606.13672

### EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments
- 类型：论文
- 方向：Agent
- 发布日期：2026-06-11
- 来源与作者：Jundong Xu、Qingchuan Li、Caiming Xiong、Bryan Hooi、Zhiyuan Hu 等
- 一句话结论：长期 Agent 的记忆应记录环境如何变化，而不只是保存静态事实。
- 核心内容：EvoArena 将终端、软件和社交环境表示为连续更新序列；EvoMem 用 patch history 保存记忆演化。现有 Agent 平均准确率仅 39.6%；EvoMem 在 EvoArena 提升 1.5%，在 GAIA、LoCoMo 分别提升 6.1% 和 4.8%。
- 创新点：动态环境评测；结构化 memory patch；连续演化链指标。
- 为什么值得关注：更贴近软件升级、规则变化和用户偏好漂移等真实场景。
- 局限与疑点：主基准增益有限；错误更新、patch 冲突和长期压缩成本尚未解决。
- 开源情况：摘要未提供可核验链接。
- 原始链接：https://arxiv.org/abs/2606.13681

### AgentBeats: Agentifying Agent Assessment for Openness, Standardization, and Reproducibility
- 类型：论文 / 开源平台
- 方向：Agent
- 发布日期：2026-06-11
- 来源与作者：Xiaoyuan Liu、Jianhong Tu、Yuqi Chen、Dawn Song 等
- 一句话结论：将 benchmark 实现为 Judge Agent，并使用 A2A 与 MCP，可降低不同 Agent 的评测集成成本。
- 核心内容：Agentified Agent Assessment 将评测逻辑与被测 Agent 分离，使理论集成量由 N×M 降至 N+M。五个月竞赛覆盖 298 个 Judge Agent、12 类任务及 467 个 Subject Agent。
- 创新点：Judge Agent 范式；A2A+MCP 评测协议；支持开放、隐私和复现约束的多种模式。
- 为什么值得关注：Agent 评测正转向协议化、生产一致的系统级评估。
- 局限与疑点：Judge Agent 会增加成本并可能产生语义漂移；接口统一不代表评分标准一致。
- 开源情况：[AgentBeats 平台](https://agentbeats.org)
- 原始链接：https://arxiv.org/abs/2606.13608

## 今日趋势观察

Agent 研究的增量正从单纯扩大模型转向接口与环境工程：SpatialClaw 优化动作接口，HyperTool 调整执行粒度，AgentBeats 标准化评测协议，EvoArena 则引入动态记忆。多模态研究也更强调长程闭环，而非单次生成。值得持续跟踪持久化执行状态、MCP/A2A、动态记忆，以及世界模型在真实硬件上的收益。

## 今日必读

1. **SpatialClaw**：方法直指 Agent 系统设计的关键变量，并提供跨 20 个基准、六个 backbone 的系统验证。
2. **HyperTool**：对 MCP 工具生态有直接工程价值，同时完整暴露安全、可解释性及过早终止等代价。
